프로젝트 : 봉사 지원 회사

- 회원가입 등급에 관하여..
　회사 사람들에게는 '직급'의 등급을 그대로 부여
　직급이 없는 사람들에게는 '일반'의 등급을 부여

- 회사명단테이블의 등급 즉, 직급은 중복이 가능(unique하지 않음)
　이러면 회원가입의 등급에서 회사명단테이블의 직급을 외래키로 참조 불가능...

- 회원가입 진행시에 회사명단테이블의 주민번호를 참조...
　주민번호가 명단에 있는 경우에 해당 직급을 가져올수가 있나?

- 회원가입 테이블을 다른 테이블과 join하여 null값을 찾으려면, 이미 해당관련 값이 들어가있는 테이블이 존재해야함..


포함될 내용
1. 회원가입(member) 테이블 : 이름(name) / 주민번호(jumin) / 성별(gender) / 아이디(id) / 비밀번호(pwd) / 등급(grade)
등급 : 일반 사원 대리 과장 사장
< 회사명단 테이블의 직급을 참조하되, null이면 일반으로..(함수사용) → 처음에는 일반, 나중에 join으로 해당 주민번호의 직급이 있으면 등급으로 적용 >


2. 회사명단(company) 테이블 : 이름(name) / 주민번호(jumin) / 성별(gender) / 부서(team) / 사번(num) / 직급(grade) / salary(월급)
사번 예시 : 년년월월-nn → 2403-01 2403-02
직급 : 사원 대리 과장 사장
< 회원가입을 하고, 회사 사람일 경우에는 하루 뒤부터 게시글을 작성 및 조회의 권한이 바뀔 수 있다. >


3. 카테고리(category) 테이블 : 카테고리(cate) / 공개여부(secret)
카테고리 : 봉사 칭찬 자유게시판 건의사항 익명
< 봉사,칭찬,자유게시판은 공개여부 Y, 건의사항,익명은 공개여부 N >


* 공개글(Y) * → 게시글 번호는 시퀀스 사용
4. 봉사(bongsa) 테이블 : 게시글번호(no) / 카테고리(cate) / 작성자(name) / 아이디(id) / 봉사명(bname) / 봉사일(bdate) / 봉사장소(bplace) / 신청여부(sub)
<신청여부가 null이면 봉사 일정 공지로 구분!! >
5. 칭찬(good) 테이블 : 게시글번호(no) / 카테고리(cate) / 작성자(name) / 아이디(id) / 추천하는사람(gname) / 사번(num) / 내용(comment)
6. 자유게시판(free) 테이블 : 게시글번호(no) / 카테고리(cate) / 작성자(name) / 아이디(id) / 제목(title) / 내용(comment)


* 비밀글(N) * → 게시글 번호는 시퀀스 사용
< 회원 등급별로 조회할 수 있는 범위 다르도록 설정(사원 대리 과장 사장)
7. 건의사항(plus) 테이블 : 게시글번호(no) / 카테고리(cate) / 작성자(name) / 아이디(id) / 제목(title) / 내용(comment)
8. 익명(noname) 테이블 : 게시글번호(no) / 카테고리(cate) / 제목(title) / 내용(comment)


* 이달의 회원(VIP) * → (join)으로 생성 예정
	 → 회사 단체 봉사활동 참여 횟수 
	 　　// 칭찬 추천 많은 순 count(*) group by (아이디,추천사람이름) 1이상이면 1번만
	 → 이달의 회원으로 선정된 사람이 속한 직급의 직급별 평균 월급의 20% 보너스 지급!! 

9. 봉사참여명단(bongsa_checklist) 테이블 : 봉사게시글번호(bongsa_no) / 아이디(id) / 사번(num)
< 회원 테이블의 기본키, 참조할 외래키로 참여명단 만들기 >
< 회사 사람이 아니더라도 봉사에 참여할 수 있도록 설정! >

→ 함수 : 연차별로 선물을 지급하기위한 함수(현재시점 - 사번(입사시점))
→ 트리거 : 나중에 해보기!!

0. 노쇼(noshow) 테이블 : 봉사테이블과 봉사참여명단테이블을 조인하여 하나의 논리적인 테이블인 뷰(view)로 만들기
< 해당 테이블에 포함되는 사람들은 사장님 면담 대상 테이블 count>=2 월급 20% 삭감(연정)
→ 봉사일정 : select
→ 선물명단
